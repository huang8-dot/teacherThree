# 方向指导教学游戏开发需求文档

## 项目背景

用户需要一个简单的模拟游戏，用于教授英文方向指示表达，如"左转"、"右转"和"直走"。游戏将帮助英语水平不高的学生通过互动方式学习这些表达。

## 核心功能需求

### 游戏机制

1. **地图系统**：

   - 包含城镇中的各种地点，如医院、学校、公园、购物中心、博物馆等
   - 地图上清晰标记各个地点的名称和位置
   - 使用网格系统表示街区结构
   - 不能穿过墙，不能越界，道路和建筑划分清晰

2. **角色移动**：

   - 玩家可以通过按钮控制角色移动
   - 左转按钮：角色向左转向
   - 右转按钮：角色向右转向
   - 直走按钮：角色向当前方向前进
   - 支持键盘控制（方向键）

3. **随机终点系统**：

   - 游戏开始时，系统从地图地点库中随机选择一个作为终点
   - 终点信息清晰显示在界面上
   - 当角色到达终点时，系统自动判定完成任务
   - 完成后自动生成新的随机终点并开始新一轮

4. **计分系统**：

   - 根据到达终点的时间和移动步数计算得分
   - 步数越少，时间越短，得分越高
   - 连续完成任务可获得额外奖励分数
   - 显示当前得分和历史最高分

5. **游戏难度**：

   - 初级：地图地点较少，距离较近
   - 中级：地图地点适中，距离中等
   - 高级：地图地点较多，可能需要更复杂的路线规划

6. **游戏模式**：
   - 单人模式：独立完成任务，挑战个人最佳成绩
   - 双人模式：两名玩家轮流移动，比赛谁先到达终点
   - 计时模式：在限定时间内完成尽可能多的任务

### 游戏模式

1. **单人模式**：单个学生可以独立玩游戏
2. **双人模式**：两个学生可以一起玩游戏（竞争性或合作性）

### 用户界面

1. **简洁明了的控制界面**：三个主要按钮（左转、右转、直走）
2. **地图显示**：清晰展示城镇地图和角色位置
3. **地点标识**：明确标注各种城镇地点
4. **方向指示**：显示角色当前面向的方向

### 技术要求

1. **前端技术栈**：HTML, CSS, JavaScript
2. **响应式设计**：确保在 iPad 等平板设备上能正常运行
3. **简单易操作**：适合英语水平不高的学生使用

## 设计需求

### UI/UX 设计

1. **色彩方案**：使用明亮、友好的颜色，适合教育环境
2. **图标设计**：使用直观的图标表示不同的操作和地点
3. **交互设计**：简单的点击操作，即时反馈

### 地图设计

1. **网格布局**：使用简单的网格表示街区
2. **地点图标**：为医院、学校等地点设计简单的图标
3. **角色设计**：简单易识别的角色形象

## 开发计划

### 阶段一：设计 UI 原型

1. 创建游戏界面的线框图
2. 设计地图布局和地点图标
3. 设计按钮和控制界面

### 阶段二：基础功能开发

1. 搭建 HTML 结构
2. 实现 CSS 样式
3. 开发地图渲染功能

### 阶段三：交互功能开发

1. 实现角色移动逻辑
2. 开发单人游戏模式
3. 实现双人游戏模式

### 阶段四：测试与优化

1. 在 iPad 上测试响应式设计
2. 优化用户体验
3. 修复可能的 bug

## 预期成果

一个简单、有趣、教育性强的方向指导教学游戏，帮助学生在互动中学习英语方向表达，同时适应课堂教学环境和学生的英语水平。
